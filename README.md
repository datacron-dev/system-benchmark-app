# System Benchmark Dashboard

A responsive web dashboard built with **Next.js** for stress-testing local LLM inference with **llama-benchy**, including real-time GPU/system monitoring and benchmark visualization.

## ⚡ One-Line Install

```bash
curl -fsSL https://raw.githubusercontent.com/datacron-dev/system-benchmark-app/main/scripts/install.sh | bash
```

This will:
- Clone (or force-update) the repository to `~/system-benchmark-app`
- Check and auto-install/upgrade Node.js 20 LTS via nvm if needed
- Install all npm dependencies (with `--legacy-peer-deps`)
- Set up Prisma database (if `DATABASE_URL` is set)
- Create a **GNOME desktop launcher** (`System Benchmark`)
- Auto-start the app and open your browser at `http://localhost:8585` on first install

### Manual Setup

```bash
git clone https://github.com/datacron-dev/system-benchmark-app.git
cd system-benchmark-app
npm install --legacy-peer-deps
```

## ▶️ Start the Dashboard

```bash
./scripts/start.sh
```

Or directly with npm:

```bash
npm run dev      # Development mode on port 8585
npm run start    # Production mode on port 8585 (after npm run build)
```

Then open: [http://localhost:8585](http://localhost:8585)

## 🖥️ Desktop Launcher

After installation, a **System Benchmark** desktop shortcut is created automatically:
- **Location**: `~/Desktop/System_Benchmark.desktop`
- **Click** to launch the app and open your browser
- Also available in your GNOME application menu
- Uses `favicon.svg` as the icon

To recreate the shortcut manually, re-run the installer.

### ⚠️ Framework Constraints & Workarounds

Next.js, PostCSS, and TypeScript **require** certain config files at the project root:
- `next.config.js` — loaded automatically by Next.js from the working directory
- `tsconfig.json` — TypeScript resolves this from the project root
- `postcss.config.js` — PostCSS resolves from CWD
- `tailwind.config.ts` — Referenced by PostCSS/Tailwind

**Solution**: The canonical configuration lives in `config/`. Thin root-level wrappers
re-export or extend the config files:
- `next.config.js` → `require('./config/next.config.js')`
- `postcss.config.js` → `require('./config/postcss.config.js')`
- `tailwind.config.ts` → `export { default } from './config/tailwind.config'`
- `tsconfig.json` → `{ "extends": "./config/tsconfig.json" }`

Similarly, `package.json` must remain at the project root for npm/Node.js to work.
A mirror copy exists in `scripts/` for organizational reference.
